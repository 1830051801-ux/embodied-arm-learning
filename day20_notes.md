# Day 20 笔记：图像输入策略 / Vision to Action

前面大多数策略输入是坐标：

```text
object_x, object_y, hand_x, hand_y
```

但真正的 VLA/视觉机器人策略通常会直接吃图像或图像特征：

```text
image + language + robot_state -> action
```

今天先学最小版本：

```text
小图像 + hand_x/hand_y -> move_dx/move_dy
```

## CNN 是什么

CNN = 卷积神经网络。

它常用于图像特征提取：

```text
图像像素 -> 图像特征
```

然后策略头再输出动作：

```text
图像特征 + 机器人状态 -> 动作
```

## 今天的小图像

我们生成 16x16 的灰度图。

```text
白点位置 = 物体位置
```

模型要从图像里判断白点在哪里，然后输出机械臂应该往哪里动。

## 和 YOLO 的区别

YOLO路线：

```text
图像 -> YOLO -> bbox/类别 -> 坐标 -> 策略
```

端到端视觉策略路线：

```text
图像 -> CNN/视觉编码器 -> 策略 -> 动作
```

真实项目可以两条路都懂：

```text
工程第一版用 YOLO 更稳。
学习型策略/VLA 会更多使用图像特征。
```

## 今天你必须能说出来

```text
1. 图像可以作为策略输入。
2. CNN 负责从图像提取特征。
3. 视觉特征可以和机械臂状态拼接后输出动作。
4. VLA 的 Vision 部分本质上就是更强的视觉编码。
```
