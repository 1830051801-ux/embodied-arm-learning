# Day 0 - 先把路线搞清楚

这门课不是让你背 PyTorch API，也不是先学一堆数学。

目标只有一个：让你能把 PyTorch 用到你的机械臂项目里，理解它在

```text
看到物体 -> 算出坐标 -> 决策怎么动 -> 控制机械臂 -> 记录数据 -> 继续训练
```

这条链路里到底负责哪一段。

## 你的项目链路

你的机械臂项目可以拆成 6 个模块：

```text
1. YOLO
   输入：摄像头图像
   输出：物体类别、框的位置、置信度
   作用：告诉系统“看到了什么、在哪里”

2. 相机标定
   输入：YOLO 框中心的像素坐标
   输出：桌面/机械臂坐标
   作用：把图片里的点变成机械臂能理解的真实坐标

3. PyTorch 策略模型
   输入：物体类型、物体坐标、机械臂当前状态、任务指令
   输出：下一步动作，或者抓取参数
   作用：学习“当前情况应该怎么做”

4. IK / 运动学
   输入：目标位置
   输出：关节角
   作用：把“去这个点”变成每个舵机/电机该到多少角度

5. 控制板/串口/ROS
   输入：关节角或电机命令
   输出：真实机械臂运动
   作用：让硬件执行

6. 日志和数据
   输入：每次抓取的状态、动作、是否成功
   输出：训练数据
   作用：让模型越学越像一个会抓东西的系统
```

## 为什么 YOLO 不够

YOLO 只回答：

```text
这是什么？
它在图像哪里？
可信度多少？
```

YOLO 不回答：

```text
夹爪从哪个方向接近？
先抬高手臂还是先横移？
物体太靠边时怎么避开？
抓杯子和抓盒子动作是否一样？
上一次失败后这次要不要调整？
```

这些才是 PyTorch 策略模型要学的东西。

## 为什么不是一上来就接真机械臂

真实机械臂会撞桌子、夹坏东西、打到限位，还可能因为坐标错了乱动。

所以正确顺序是：

```text
先在脚本里学懂张量和训练
再在仿真/简化模型里验证策略
再 dry-run 打印命令
最后才发给真实机械臂
```

## 你今天要掌握的 3 句话

```text
PyTorch 不是一个单独的软件，它是训练模型的 Python 工具库。
YOLO 负责看见物体，PyTorch 负责学习怎么决策。
训练机械臂，本质是训练 state -> action 这个映射。
```
