# Day 6 笔记：怎么判断模型有没有学会

训练完模型以后，不能只说：

```text
脚本跑完了，所以模型会了。
```

要看证据：

```text
loss 有没有下降
测试误差是多少
不同物体上的误差是否合理
是否比简单规则更好
```

## loss 是什么

loss 是模型预测和正确答案之间的差距。

```text
loss 大：模型错得多
loss 小：模型更接近正确答案
```

但是只看训练 loss 不够。

## 为什么要看测试误差

训练数据是模型见过的。

测试数据是模型没见过的。

如果模型只在训练数据上好，在新数据上很差，就说明它没有真正学会规律。

所以更重要的是：

```text
test error
```

## 为什么按物体类别看误差

机械臂抓不同物体，策略可能不同：

```text
cup：中等夹爪宽度
bottle：夹爪窄一点
box：夹爪宽一点
```

如果只看总平均误差，可能掩盖问题。  
按类别看，能发现模型是不是某一种物体抓不好。

## 为什么比较规则和 PyTorch

你的 YOLO + 规则方案是可以做第一版的。

但是规则通常是：

```text
不管 cup/bottle/box，都按同样比例靠近
```

学习策略可以从数据里学：

```text
bottle 靠近慢一点
box 靠近直接一点
不同物体夹爪宽度不同
```

这就是 PyTorch 的价值。

## 今天你必须能说出来

```text
1. loss 下降说明模型在拟合训练数据。
2. 测试误差说明模型对新数据有没有用。
3. 按物体类别评估能发现局部问题。
4. 规则能起步，PyTorch 能从数据里学更复杂差异。
```
