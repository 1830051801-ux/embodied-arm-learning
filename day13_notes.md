# Day 13 笔记：规则策略 vs 学习策略

你的第一版机械臂可以这样做：

```text
YOLO -> 坐标 -> 写死规则 -> IK -> 控制
```

这是正确的工程起步。

但如果你要往具身智能方向讲，就要证明：

```text
学习策略比简单规则能覆盖更多情况。
```

## 什么是规则策略

例子：

```text
永远朝物体移动 45% 的距离
根据物体类别设置夹爪宽度
```

优点：

```text
简单
稳定
容易调试
数据少也能跑
```

缺点：

```text
复杂场景要写很多 if/else
不同物体、不同角度、不同遮挡很难覆盖
```

## 什么是学习策略

学习策略从数据里学：

```text
状态 -> 动作
```

如果数据里体现了：

```text
bottle 要慢一点靠近
box 要更直接靠近
cup 用中等夹爪宽度
```

模型就可能学到这些差异。

## 为什么要有 baseline

baseline 就是对照组。

你不能只说：

```text
我用了 PyTorch。
```

你要能说：

```text
我和规则策略做了对比，PyTorch 在测试集上误差更低。
```

## 今天你必须能说出来

```text
1. YOLO+规则是好起点，不是错误方案。
2. PyTorch策略的价值要靠指标证明。
3. baseline对比是项目里很重要的实验。
4. 面试时要讲“为什么学习策略更好”，不是只讲“我用了AI”。
```
