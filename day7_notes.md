# Day 7 笔记：接真实机械臂前的 dry-run

前面我们已经有完整链路：

```text
YOLO -> 标定 -> PyTorch策略 -> IK -> 关节角
```

Day 7 要回答：

```text
怎么算出来的关节角，怎么变成控制板命令？
```

但注意：

```text
今天只 dry-run，只打印命令，不发送给电机。
```

## 为什么不能直接发给真实机械臂

真实机械臂有风险：

```text
角度超限
方向反了
速度太快
夹爪夹坏东西
撞到桌面
串口协议写错
```

所以第一步必须是：

```text
只打印命令 -> 人看懂 -> 检查范围 -> 低速小角度测试
```

## dry-run 命令长什么样

这套课程用教学命令格式：

```text
J q1 q2 G gripper_mm
```

例子：

```text
J -52.39 163.53 G 34.4
```

含义：

```text
q1 = -52.39 度
q2 = 163.53 度
gripper = 34.4 mm
```

真实项目里要替换成你的控制板协议。

## 为什么要做限幅

IK 算出来的角度不一定安全。

所以要限制：

```text
q1 在 -90 到 90 度
q2 在 0 到 170 度
gripper 在 20 到 100 mm
```

真实机械臂要用你自己的关节限位。

## 接真实机械臂前必须确认

```text
1. COM 口
2. 波特率
3. 命令格式
4. 每个关节角度范围
5. 夹爪命令格式
6. 急停/断电方式
7. 是否支持低速模式
```

## 今天你必须能说出来

```text
1. dry-run 是只打印不发送。
2. IK 输出角度后还要检查安全范围。
3. 课程里的 J q1 q2 G gripper_mm 只是教学格式。
4. 真实项目必须按控制板协议改。
5. 接真实电机前要先低速、小角度、可急停。
```
