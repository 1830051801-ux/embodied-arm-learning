# Day 10 笔记：串口 dry-run

前面 Day 7 生成了教学命令：

```text
J -52.39 163.53 G 34.4
```

Day 10 要回答：

```text
这条命令以后怎么发给真实控制板？
```

但今天仍然不发给电机，只 dry-run。

## 为什么继续 dry-run

真实机械臂控制板需要确认：

```text
COM 口
波特率
命令格式
换行符
关节角范围
夹爪单位
是否需要校验位/帧头/帧尾
急停方式
```

这些没确认前，不能直接发送。

## 串口发送通常需要什么

```text
port = COM3
baudrate = 115200
command = "J -52.39 163.53 G 34.4\n"
```

发送逻辑通常是：

```text
打开串口
写入命令 bytes
等待控制板响应
关闭串口
```

## dry-run 模板的意义

我们先写成：

```text
dry_run=True
```

它只打印：

```text
DRY_RUN_SEND: J -52.39 163.53 G 34.4
```

确认格式没问题以后，才考虑：

```text
dry_run=False
```

## 今天你必须能说出来

```text
1. 串口命令要有明确协议。
2. 不知道协议时不能直接发给电机。
3. dry-run 是安全验证命令格式。
4. 真机前要先小角度、低速、可急停。
```
