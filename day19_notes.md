# Day 19 笔记：语言条件策略 / VLA 最小概念

前面的策略输入主要是：

```text
物体坐标
机械臂状态
物体类别
```

VLA 多了一个重要条件：

```text
语言指令
```

VLA = Vision-Language-Action。

可以理解成：

```text
视觉看到什么 + 语言要求做什么 -> 机器人动作
```

## 为什么需要语言

同一个场景：

```text
物体在 x=0.3, y=0.1
手在 x=0.0, y=0.0
```

不同指令：

```text
pick：去抓它
push：推它
pull：拉它
```

动作应该不一样。

所以策略输入不能只有坐标，还要有任务意图。

## 最小语言编码

今天不用大语言模型，只用 one-hot：

```text
pick -> [1, 0, 0]
push -> [0, 1, 0]
pull -> [0, 0, 1]
```

这和物体类别 one-hot 是同一个思想。

## 真正 VLA 更复杂在哪里

今天：

```text
小词表 one-hot + 坐标 -> 动作
```

真正 VLA：

```text
图像编码 + 语言模型/文本编码 + 机器人状态 -> 动作序列
```

但核心关系一样：

```text
language changes action
```

## 今天你必须能说出来

```text
1. VLA 里的 L 是语言条件。
2. 同一个视觉状态，不同语言指令可以对应不同动作。
3. 最小实现可以先用 one-hot 表示指令。
4. 真正 VLA 会用更强的视觉和语言编码器。
```
